# BMAD Method: Agile AI Driven Development

This project follows the **BMAD Method** (Breakthrough Method for Agile AI Driven Development).

## Core Philosophy
- **Agentic Workflow**: Specialized AI agents work in concert (PM, Architect, Developer, QA).
- **Phased Approach**: Analysis -> Planning -> Solutioning -> Implementation.
- **Artifact-Driven**: Every phase produces structured artifacts (markdown files).

## Workflow Phases

### 1. Analysis (Optional)
- **Goal**: Brainstorm, research, and explore solutions.
- **Output**: `_bmad-output/planning-artifacts/`

### 2. Planning
- **Goal**: Define what to build.
- **Actions**: Create PRDs (Product Requirement Documents) and Tech Specs.
- **Output**: `_bmad-output/planning-artifacts/prd.md`

### 3. Solutioning
- **Goal**: Design how to build it.
- **Actions**: Define Architecture and UX/UI.
- **Output**: `_bmad-output/planning-artifacts/architecture.md`, `_bmad-output/planning-artifacts/ux-design.md`

### 4. Implementation
- **Goal**: Build and validate.
- **Actions**: Write code, run tests, verify changes.
- **Output**: `_bmad-output/implementation-artifacts/`

## Agent Guidelines
- **Context**: Always check `AGENTS.md` and existing artifacts in `_bmad-output` before starting a task.
- **Verification**: Verify every step. Do not assume success.
- **Documentation**: Update artifacts as the project evolves.

For full documentation, visit [BMAD-METHOD on GitHub](https://github.com/bmad-code-org/BMAD-METHOD).
